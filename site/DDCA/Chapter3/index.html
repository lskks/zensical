第3章 时序逻辑设计 (Sequential Logic Design)

3.1 概述

  • 组合逻辑：输出仅取决于当前输入
  • 时序逻辑：输出取决于当前和过去的输入（具有记忆功能）
  • 状态 (State)：系统过去信息的浓缩，用于决定未来行为
  • 状态变量：存储状态的二进制位

3.2 锁存器与触发器 (Latches and Flip-Flops)

3.2.1 基本存储单元 - 双稳态元件

  • 交叉耦合反相器：两个反相器首尾相连，具有两个稳定状态（Q=0 或 Q=1）
  • 双稳态元件存储 1 bit 信息
  • 亚稳态 (Metastable State)：两个输出约在 VDD/2 的中间状态，是不稳定状态
  • 上电时初始状态未知且不可预测

3.2.2 SR 锁存器

  • 由两个交叉耦合的 NOR 门组成
  • S (Set)：将 Q 置为 1
  • R (Reset)：将 Q 置为 0
  • S=R=0：保持原状态 (Q = Qprev)
  • S=R=1：非法状态（Q 和 Qbar 均为 0）

3.2.3 D 锁存器

  • 解决了 SR 锁存器的"什么"和"何时"分离问题
  • D：数据输入（what）
  • CLK：时钟输入（when）
  • CLK=1 时透明 (transparent)：Q = D
  • CLK=0 时不透明 (opaque)：Q 保持原值

3.2.4 D 触发器 (Flip-Flop)

  • 由两个背靠背的 D 锁存器构成（主锁存器 + 从锁存器）
  • 边沿触发 (Edge-Triggered)：仅在时钟上升沿将 D 复制到 Q
  • 其余时间保持状态
  • 符号中的三角表示边沿触发时钟输入
  • 一个 D 触发器约需 46 个晶体管（用逻辑门实现），或 20 个晶体管（用传输门实现）

3.2.5 寄存器

  • N 位寄存器：N 个触发器共享同一时钟，同时更新所有位

3.2.6 使能触发器 (Enabled Flip-Flop)

  • 增加 EN 引脚控制是否在时钟沿加载数据
  • 实现方式：输入多路复用器 或 门控时钟（不推荐）

3.2.7 可复位触发器 (Resettable Flip-Flop)

  • 同步复位：仅在时钟上升沿复位
  • 异步复位：RESET 有效时立即复位（与时钟无关）

3.2.8 传输门级锁存器和触发器设计

  • 使用传输门可大幅减少晶体管数量
  • 紧凑型 D 锁存器约需 12 个晶体管（静态）
  • 传输门级 D 触发器约需 20 个晶体管

3.3 同步逻辑设计

3.3.1 有问题的电路

  • 环形振荡器 (Ring Oscillator)：奇数个反相器首尾相连，产生周期性振荡
  • 竞争条件 (Race Condition)：电路行为取决于哪条路径最快，导致不可靠

3.3.2 同步时序电路

  • 在循环路径中插入寄存器，打破循环
  • 同步时序电路定义：
  • 每个元件都是寄存器或组合电路
  • 至少有一个寄存器
  • 所有寄存器接收相同时钟
  • 每个循环路径至少包含一个寄存器

3.3.3 同步 vs 异步电路

  • 异步电路更通用但更难设计
  • 几乎所有数字系统都是同步的

3.4 有限状态机 (Finite State Machines, FSMs)

3.4.1 FSM 设计流程

  1. 识别输入和输出
  2. 绘制状态转换图
  3. 编写状态转换表和输出表
  4. 选择状态编码（二进制编码 / One-Hot 编码）
  5. 推导次态和输出布尔方程
  6. 绘制电路原理图

交通灯控制器示例：

  • 输入：TA, TB（传感器）
  • 输出：LA, LB（红/黄/绿）
  • 4 个状态：S0(绿/红) → S1(黄/红) → S2(红/绿) → S3(红/黄)

3.4.2 状态编码

编码方式 状态数 K 所需位数 优点 缺点
二进制编码 log₂K 触发器少 组合逻辑复杂
One-Hot 编码 K 组合逻辑简单 触发器多

3.4.3 Moore 机 vs Mealy 机

特性 Moore 机 Mealy 机
输出依赖 仅当前状态 当前状态 + 输入
输出标在 圆圈内 转换弧上
状态数 通常较多 通常较少
输出响应 慢一个周期 立即响应

3.4.4 状态机分解 (Factoring)

  • 将复杂 FSM 分解为多个简单交互的 FSM
  • 应用层次化和模块化思想

3.5 时序逻辑的时序分析

3.5.1 动态纪律 (Dynamic Discipline)

  • 触发器的输入必须在建立时间 (tsetup) 和保持时间 (thold) 内保持稳定
  • 孔径时间 (Aperture Time) = tsetup + thold

3.5.2 系统时序

  • 建立时间约束 (Setup Time Constraint)：
  • Tc ≥ tpcq + tpd + tsetup

  • 决定最小时钟周期（最大工作频率）

  • 保持时间约束 (Hold Time Constraint)：

  • tccq + tcd ≥ thold
  • 违反后无法通过调整时钟周期修复（需重新设计电路）

3.5.3 时钟偏移 (Clock Skew)

  • 时钟到达不同寄存器的时间差异
  • 建立时间约束：Tc ≥ tpcq + tpd + tsetup + tskew
  • 保持时间约束：tccq + tcd ≥ thold + tskew
  • 时钟偏移使时序约束变差

3.5.4 亚稳态 (Metastability)

  • 输入在孔径时间内变化时，输出可能处于 VDD/2 附近
  • 最终会解析为 0 或 1，但解析时间无上界
  • 概率服从指数分布：P(tres > t) = (T0/Tc) × e^(-t/τ)

3.5.5 同步器 (Synchronizer)

  • 用于处理异步输入
  • 两级触发器同步器：两个背靠背触发器，等待一个时钟周期让亚稳态解析
  • MTBF (平均无故障时间)：MTBF = e^(Tc/τ) / (N × T0/Tc)
  • MTBF 随等待时间指数级增长

3.6 并行性 (Parallelism)

  • 延迟 (Latency)：一个任务从开始到结束的时间
  • 吞吐量 (Throughput)：单位时间内处理的任务数
  • 空间并行：复制多个硬件副本同时工作
  • 时间并行（流水线 Pipelining）：将任务分解为多个阶段，不同任务在不同阶段重叠执行
  • 流水线通过在组合逻辑间插入寄存器来实现
  • 流水线可大幅提高吞吐量，但会略微增加延迟

3.7 本章总结

  • 时序逻辑输出取决于当前和过去的输入，具有记忆功能
  • 触发器是最重要的时序元件，在时钟上升沿采样
  • 同步时序电路由组合逻辑块 + 时钟寄存器构成，易于设计
  • FSM 是设计时序电路的系统性方法
  • 时序约束决定系统最大工作频率
  • 亚稳态不可避免，需通过同步器管理
  • 并行性（尤其是流水线）是提高系统性能的关键技术
